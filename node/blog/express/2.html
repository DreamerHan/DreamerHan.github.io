# 中间件

# express 的中间件模式

先上示例代码

每一个 use 方法执行的就是一个中间件，next 开启执行后续程序，当后续的程序执行结束后，才会执行当前中间件 next 的后续代码

const express = require("express");

const app = express();

// 请求开始整体拦截
app.use(function (req, res, next) {
  console.log("服务接受请求开始了...");

  next();

  console.log("next 之后 -- 服务请求结束了...");
});

// cookie 模拟
app.use(function (req, res, next) {
  console.log("cookie 模拟存入 req.body");
  req.cookie = {
    userid: "hello world",
  };

  next();

  console.log("next 之后 -- Cookie 模拟 ");
});

// 接口入口
app.use("/api", function (req, res, next) {
  console.log("/api 开头的接头请求进入");
  next();
});

// get 请求，路由请求的另一种写法
app.get("/api/getInfo", function (req, res, next) {
  console.log("GET请求 /api/getInfo 的获取");
  res.json({
    errno: 0,
    data: "GET OK",
  });
});

// post 请求，路由请求的另一种写法
app.post("/api/postInfo", function (req, res, next) {
  console.log("POST请求 /api/getInfo 的获取");
  res.json({
    errno: 0,
    data: "POST OK",
  });
});

app.listen(3000, () => {
  console.log("Server is running at port 3000");
});

所以，上述代码的执行顺序是：

  • 服务接受请求开始了...
  • cookie 模拟存入 req.body
  • /api 开头的接头请求进入
  • GET 请求 /api/getInfo 的获取
  • next 之后 -- Cookie 模拟
  • next 之后 -- 服务请求结束了...

但是 💢💢💢，express 模式的中间件碰到异步，就很尴尬了。

❓❓❓ 异步好像会阻断原来的回调顺序，不知道是不是写的有问题，或是不知道如何给 next 的后续代码写异步

碰到异步后，含有异步代码的中间件假如是 B，B 的上一个中间件加入是 A。A 中的 next 后的代码会直接调用， 而不会等 B 中的异步执行结束后以及 B 中的 next 包含的后续代码执行结束后。
