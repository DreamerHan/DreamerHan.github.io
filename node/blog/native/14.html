# crontab 拆分日志

# 基础

crontab 是一个 linux 命令，一个定时任务

  • crontab -e 开启设置 crontab 任务的文本面板
  • crontab -l 查询已经设置的 crontab 任务命令

设置定时任务格式是 ***** command

  • 第一个 * 表示 分钟，写成 * 表示忽略
  • 第二个 * 表示 小时，写成 * 表示忽略
  • 第三个 * 表示 日期（几号），写成 * 表示忽略
  • 第四个 * 表示 月份，写成 * 表示忽略
  • 第五个 * 表示 星期，写成 * 表示忽略
# 表示每1分钟执行一次 command
1 * * * * command

# 表示每天的第一个小时执行一次 command
* 1 * * * * command

# 表示每天的第一个小时的第二分钟执行一次 command
2 1 * * * command

# 日志拷贝脚本 utils/copy.sh

将 access.log 拷贝并重命名为 xxxx.access.log，清空 access.log 文件,继续积累日志

#!/bin/sh

# 进入到日志
cd /Users/hanzhizhen/code-study/pro-blog-node/server/logs

# 拷贝 access.log 到 日期.access.log 文件中
cp access.log $(date +%Y-%m-%d).access.log

#清空 access.log
echo "" > access.log

# 设置 crontab 命令


# 开启设置 crontab 命令的文本输入界面
crontab -e

# 在内部输入 crontab 的命令
# 比如：每天零点执行相关脚本，注意 sh 后面的 command 要写绝对路径
# 比如下面是当前电脑脚本位置
* 0 * * * sh /Users/hanzhizhen/code-study/pro-blog-node/server/utils/copy.sh

# 保存成功后会提醒
crontab: installing new crontab

# 查询已设置指令
crontab -l
