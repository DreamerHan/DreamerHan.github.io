# 安装一个全局命令

  1. yarn init -y 创建一个 npm 管理的项目 study-cli

  2. 在 package.json 中添加 bin 字段

{
  "name": "study-cli",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "bin": {
    "h-cli": "./cli.js"
  }
}

h-cli 就是将来的全局命令，全局执行 h-cli 后调用的就是指定的 ./cli.js 文件。

对象的写法是在处理多个可执行文件，如果只有一个可执行文件，bin 可以写成字符串的形式，但是命令的名称就必须是 package.json 的 name 了。

{
  "name": "h-cli",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "bin": "./cli.js"
}

上述写法，类似于：

{
  "name": "h-cli",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "bin": {
    "h-cli": "./cli.js"
  }
}

除了通过 bin 来指定执行文件外，还可以在 文件目录下设置 bin 目录来设置执行文件,通过设置 package.json 中的 directories.bin。

WARNING

bin 和 directories.bin 是不能同时设置的，否则会报错。单个执行文件用 bin，多个执行文件，用 directories.bin。不过多个的，暂时没尝试过。٩(●˙▿˙●)۶…⋆ฺ
