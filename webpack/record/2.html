# 关于 webpack-dev-server

# 运行原理

webpack-dev-server 库的原理是启动一个服务器，当浏览器访问这个服务时，与浏览器通过 WebSocket 进行长链接。

同时 webpack-dev-server 会开启 webpack 监听本地代码文件的功能。当本地代码文件更新之后，进行重新打包编译。

webpack-dev-server 通过 WebSocket 将更新模块信息推送给浏览器。浏览器根据此次编译信息，去获取最新代码。

TIP

  1. Webpack 本身就支持监听文件代码的变化，而 webpack-dev-server 的作用仅仅是默认开启 webpack 的这一项属性。

  2. webpack-cli@4 版本以后，启动 webpack-dev-server 的指令不再是 webpack-dev-server。而是 webpack serve

# 一些配置

webpack-dev-server 读取的属性需要配置在 webpack 配置文件的 devServer 属性下。webpack-dev-server 执行时会读取这个属性来做配置。

devServer: {
  // 启动服务的host，默认是 localhost
  host: '127.0.0.1',

  // 服务启动的端口，默认是 8080
  port: 9527,

  // 是否自动打开浏览器 string | boolean
  // 默认 fales，设置为 true 打开默认浏览器
  open: true,

  // 是否启动 gzip 压缩，默认是 false
  compress: true,

  // 是否启动热更新(HMR)，默认是 false
  // 热更新使用的是webpack中HotModuleReplacementPlugin
  hot: true,

  // 是否开启 https 请求，默认是 false
  https: true,

  // 告诉开发服务器是否启用魔法 HTML 路由（与 webpack 输出相对应的路由，例如“/main”对应“main.js”）
  maginHtml: true,

  // 检测到文件更改时启用重新加载/刷新页面（默认启用）。
  // 此属性优先级低于hot
  // 为了 liveReload 能够生效，devServer.hot 配置项必须禁用或者 devServer.watchFiles 配置项必须启用
  liveReload: true,

  // 允许通过指定的索引页面（默认为“index.html”）代理请求，这对于利用 HTML5 History API 的单页应用程序很有用。
  // 具体来说，当使用webpack-dev-server启动开发服务器时，会监听指定的端口，并将静态文件（如HTML、CSS、JavaScript等）提供给浏览器。当浏览器发起一个请求时，webpack-dev-server会根据配置的静态文件路径进行匹配，如果找到了对应的静态文件，则返回该文件。如果请求的URL没有匹配到任何静态文件，即请求的是前端路由的路径，此时history-api-fallback配置项会生效，它会将这个请求重定向到指定的HTML文件，通常是前端应用程序的入口文件，这样前端应用程序就能够正确加载，并根据URL路径来渲染相应的页面1。
  historyApiFallback: false,

  // 该选项允许将允许访问开发服务器的服务列入白名单
  // auto | all | ['host.com']
  allowedHosts: 'all',

  // 该配置项允许你配置 globs/directories/files 来监听文件变化
  watchFiles: ['src/**/*.php', 'public/**/*'],

  // 配置从目录提供静态文件的选项（默认是 'public' 文件夹）。将其设置为 false 以禁用
  // string | boolean | object | [string, object]
  static: {
    // 告诉服务器从哪里提供内容。只有在你希望提供静态文件时才需要这样做。
    // static.publicPath 将会被用来决定应该从哪里提供 bundle，并具有优先级。
    directory: path.join(__dirname, 'public'),

    // 告诉服务器在哪个 URL 上提供 static.directory 的内容
    publicPath: '/'
  }

  // 代理配置
  proxy: {
    '/api': {
      target: '', // 代理目标地址
      pathRewrite: { '^/api': '' }, // 路径重写
      changeOrigin: true
    }
  }

}
